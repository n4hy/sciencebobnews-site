# Science Bob News — site repo

This repository hosts the **rendered output** of the [Science Bob News](https://github.com/n4hy/ScienceBobNews) UAP / UFO / USO news fusion engine. It is not a place where humans commit code; it is an outbound landing pad. The engine runs elsewhere, builds a static SITREP, and pushes the finished files here. GitHub Pages then serves them at:

**🌐 https://n4hy.github.io/sciencebobnews-site/**

---

## What's in here

| File | Source | Refreshed |
|---|---|---|
| `index.html` | Generated by `render_html` in the engine | Every 10 min |
| `sitrep.md` | Generated by `render_markdown` | Every 10 min |
| `feed.json` | Generated by `render_feed` (machine-readable) | Every 10 min |
| `card_c_{1..5}.svg` | Generated by `make_story_card_svg` — one card per corroborated story | Every 10 min |
| `card_d_{1..5}.svg` | One card per developing-lane story | Every 10 min |
| `banner_*.svg` | Original section banners (currently unused on the rendered article, kept for future reuse) | Static |

Every story card is an **original SVG** composed from that story's own headline + Admiralty reliability rating + source count + platform list. No third-party imagery, no re-hosted media. The only embedded content is the engine's own metadata, rendered as geometry, text, and color.

## How the data path works

```
Bluesky firehose ─┐
Mastodon tags ────┼─► engine clusters → BitNet reliability scoring
9 podcast YT RSS ─┘                              │
                                                 ▼
                                       build/public/ on the build host
                                                 │
                                                 ▼  outbound git push
                                       this repo  ─► GitHub Pages CDN
                                                 │
                                                 ▼
                                       https://n4hy.github.io/sciencebobnews-site/
```

The build host runs the engine via cron:

- `*/10 * * * *` — full SITREP build, push here, broadcast on Bluesky + Mastodon
- `0 * * * *` — video-only SITREP, broadcast podcast video links
- `*/30 * * * *` — copy `article.md` (X Article-ready markdown) to the operator's clipboard and notify

The build is intentionally **outbound-only**: the build host opens no inbound ports and runs no web server. It pushes finished files here; GitHub serves them.

## How a fresh SITREP appears here

Each cycle the engine:

1. Pulls public free sources (no API keys): Bluesky's public firehose, Mastodon hashtag timelines, and YouTube RSS feeds for the tracked podcasts (Need to Know · Theories of Everything · Weaponized · Project Unity · American Alchemy · Area 52 · Orbis Paranormal · Fade to Black · Spaced Out Radio).
2. Runs the UAP topic gate.
3. Clusters social-feed posts to count *independent* corroboration. Podcast episodes bypass clustering so each episode is its own story.
4. Scores each event with the BitNet b1.58 reliability classifier (Admiralty letter + number).
5. Generates `index.html`, `sitrep.md`, `feed.json`, and one card SVG per story.
6. `git add -A && git commit -m "SITREP <timestamp>" && git push` — that's the only thing that touches this repo.

A push happens only when the rendered output actually changed, so an unchanged cycle costs nothing.

## Where the code lives

Engine, publisher, RSS adapters, BitNet trainer, X-Article notifier, and the card SVG generator are all in the sibling repo: **[github.com/n4hy/ScienceBobNews](https://github.com/n4hy/ScienceBobNews)**.

If you want to read the renderer that produced any file in this repo, the corresponding function is in `science_bob_news.py` there:

| File you see | Function that built it |
|---|---|
| `index.html` | `render_html(confirmed, developing, ts)` |
| `sitrep.md` | `render_markdown(...)` |
| `feed.json` | `render_feed(...)` |
| `card_*.svg` | `make_story_card_svg(story, kind)` |
| `article.md` (not committed here — operator-only) | `render_article(...)` |
